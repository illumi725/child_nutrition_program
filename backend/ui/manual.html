HAPAG Form 5A Comparator

An advanced dual-mode desktop environment designed to synchronize, optimize, and parse local nutritional data registries with a secure production backend.

System Overview

The HAPAG Form 5A Comparator is built specifically for ASA Philippines Foundation, Inc. to streamline baseline monitoring and child nutrition programs. By comparing raw Excel spreadsheets directly against an active central database, the application bridges the gap between field updates and cloud records.

Dual Connection Architecture
Works seamlessly in high-speed environment setups (Cloud Realtime) and remote field operations (Local Offline SQLite Snapshot) with automatic synchronization capability.

Security & Authentication

Access control mechanisms protecting clinical records and user privileges.

Google Single Sign-On (SSO)

The system leverages a secure, modern Google OAuth dialog window. Users can sign in using their registered organization Google account. Authentication status is visually confirmed via a successful browser landing page with a closing countdown helper.

Once authenticated, the Google email is cross-checked against authorized user profiles. If found, session caches store the address to expedite future desktop launches.

Access Verification

After a successful Google SSO handshake, the user must input their unique role-based Access Code:

  • The access code is parsed and matched with database user profiles.
  • Accounts carry granular permissions (e.g. Administrator, Standard User).
  • Security Lockout: Entering the incorrect access code 5 consecutive times automatically locks the terminal and shuts down the application safely.

Directory Scanning & Index Rebuilds

Parsing local files and identifying record matches in real time.

File Explorer Scanner

The left panel provides a quick directory browser to locate regional/site Excel spreadsheets. Selecting folders populates target files for comparison.

Clicking Scan Selected Files starts a comprehensive background parsing job using ScanWorker. The engine extracts beneficiary listings, baseline metrics, weight, height, and collection milestones.

Inter-file Duplicate Index

To accelerate scans and prevent inter-file duplication, the scanner creates a locally cached index file (interfile_index_cache.json) tracking mapped rows across documents.

Rebuilding Index
If spreadsheets are modified or renamed externally, click the Rebuild Inter-file Index button. This clears the cache and constructs a clean comparison index.

Interactive Conflict Resolver

Detailed breakdown of the comparison discrepancy tabs.

Matching and Correction Grids

The main screen classifies findings into nine specialized, tabbed results views. Each grid lists conflicting data side-by-side with clear action controls. The tables feature horizontal scrolling to ensure full visibility of long names and file paths without truncation.

Grid Tab Description Resolving Actions
Exact Matches All demographics are identical, but baseline height/weight has newer updates in the Excel sheet. Click Sync to update the database immediately, or use Bulk Sync All.
High Confidence Name similarity scores indicate highly likely matches with minor database variations. Review and match baseline entries to align records.
Review Required Significant parameter differences require manual human confirmation. Double-check original papers before synchronizing.
Name Discrepancies Birthdays and baselines match, but name spelling differs (e.g., misplaced middle names). Select Use Excel to update the DB, Use DB to update Excel, or type manually.
Birthday Discrepancies Identical names mapped but birthdays disagree (e.g., typos in input fields). Choose Use Excel, Use DB, or input a Manual Date to correct both files.
Missing in DB Beneficiaries found in the Excel sheet who do not exist in the database. Click Add to DB to open the registration dialog and record the child.
Missing in Excel Beneficiaries existing in the database but completely absent from the scanned Excel baseline. Click Delete from DB to securely remove outdated or erroneous records permanently.
Excel Duplicates Multiple entries found with the exact same name within the scanned Excel files. Click Manage to view the conflicts and decide which row is valid.
DB Duplicates Multiple records found with the same name residing inside the database itself. Click Manage to review related records and resolve these internal database redundancies.

Database Duplicate Beneficiary Manager

Resolving multiple database records to maintain index integrity.

DB Duplicate Records

When multiple records are detected with overlapping identifiers or identical names, standard queries are hindered, and metrics can split across entries.

The system opens the Manage DB Duplicate Beneficiaries dashboard, which pulls diagnostic statistics (e.g., Feeding schedules, Baseline metrics, Height/Weight records, Absences) for each copy from the database.

Select Column Checkboxes

The left-most column contains Select Checkboxes. Clicking a row or checking its box selects that record, highlighting its details in red on the action bar.

  • Keeper Highlighting: The row containing the most recorded feeding days or baseline details is automatically highlighted in Green as the recommended record to preserve.
  • Cascade Hard Delete: Clicking Delete Selected Record prompts a warning confirmation, listing all child rows that will be safely deleted. It then deletes that duplicate and syncs child records to the survivor.

Cloud & Offline Synchronization

Configuring local operations and server delta uploads.

Database Mode Settings

Accessible from the settings menu:

  • Cloud Mode: Realtime, direct MySQL production link. Requires persistent internet.
  • Local Mode: Runs on a local SQLite file stored in AppData. Ideal for offline work.

Mode Swap Restarts
Switching database modes automatically restarts the application to securely rebuild connections and clear memory footprints.

Local Sync Options

Under Local Mode, sync controls include:

  • Full Replication: Executed during first-time setup or database reset. It completely recreates SQLite tables from live MySQL schemas, establishes optimized database indexes, and pulls all cloud data.
  • Delta Sync Now: Performs bidirectional delta updates. It pushes local modifications (based on updated_at timestamps) via MySQL ON DUPLICATE KEY UPDATE, and pulls cloud records.
  • Auto-Sync Timer: Enables background delta synchronization at user-configured minute intervals.

System Dashboard & Global Search

Quick statistics, analytics representation, and records indexing.

System Dashboard

Accessed via the header's 📊 System Dashboard button, this utility computes real-time statistics:

  • Total database and regional sites count.
  • Offline snapshot integrity status.
  • Active synchronization indicators.
  • Detailed feeding logs charts and progress summaries.

Global Beneficiary Search

The 🔍 Search Beneficiary tool performs case-insensitive wildcard searches of the local database by First Name and Last Name.

It instantly retrieves registered demographics, including birthdates, site names, and exact counts of related baseline parameters (Feedings, Absences, Height/Weight histories). If duplicate profiles are detected under that name, it displays a button to open the DB Duplicate Beneficiary Manager.
